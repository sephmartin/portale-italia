Project Brief · Aprile 2026

CittadinoOS

Portale Italia

Un sistema operativo per il cittadino italiano

GitHub  ·  portale-italia.online

🔥 Il problema

  • PA frammentata in silos: INPS, Entrate, PagoPA, ANPR, Fascicolo Sanitario, Portale Automobilista
  • Ogni ente: interfaccia, login, logica separate
  • Il cittadino fa da middleware umano
  • Non è tecnico — le informazioni esistono. È un problema di design.

"Nessuno ha mai progettato l'esperienza complessiva del cittadino"

💡 La soluzione

  • Unico punto di accesso a tutti i servizi PA
  • Autenticazione SPID/CIE unificata
  • Layer intelligente sopra i sistemi esistenti
  • ispirato a Mint.com, ma per la burocrazia italiana

🚀 Il prototipo

  • Sidebar unificata per tutti i moduli PA
  • Dashboard con notifiche aggregate
  • Agente vocale per modulo (Web Speech API)
  • API Gateway OpenAPI 3.0, 9+ endpoint
  • Design Design System Italia (token ufficiali), WCAG 2.1 AA
  • Dark mode, mobile responsive, drawer navigation

Non è un mockup — è codice che gira.

🎯 La visione

Automazione fiscale/finanziaria via PSD2:

  • Pre-compilazione automatica 730, Unico, F24
  • Notifiche proattive per movimenti bancari
  • Zero dichiarazioni manuali

Obiettivo: rendere la PA invisibile.

  • Accessibilità: WCAG 2.1 AA, GLI AGID
  • Approccio: AI-native, vibe coding

❓ Le domande aperte

Interoperabilità

API PA non aperte. Soluzione: middleware + PSD2 + autorizzazione utente. EUDI Wallet come layer identità.

Privacy

Agente on-device, zero storage server-side, SPID nativa.

Accessibilità

Voce come interfaccia primaria, zero gergo tecnico, unica login.
