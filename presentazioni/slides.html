CittadinoOS

Portale Italia

Un sistema operativo per il cittadino italiano

Project Brief v4 · Aprile 2026

Repo | Demo

🔥 Il problema

  • PA frammentata in silos: INPS, Entrate, PagoPA, ANPR, Fascicolo Sanitario...
  • Ogni ente: interfaccia, login, logica separate
  • Il cittadino fa da middleware umano
  • Non è tecnico — le informazioni ci sono. È un problema di design.

💡 La soluzione

  • Unico punto di accesso a tutti i servizi PA
  • Autenticazione SPID/CIE unificata
  • Layer intelligente sopra i sistemi esistenti
  • ispirato a Mint.com, ma per la burocrazia italiana

🚀 Il prototipo

Web app React con:

  • Sidebar unificata per tutti i moduli PA
  • Dashboard centralizzata con notifiche aggregate
  • Agente vocale per modulo (Web Speech API)
  • API Gateway OpenAPI 3.0, 9+ endpoint
  • Design system Design System Italia (token ufficiali), WCAG 2.1 AA
  • Dark mode, mobile responsive, drawer navigation

Non è un mockup — è codice che gira.

🎯 La visione

Automazione fiscale e finanziaria via PSD2:

  • Pre-compilazione automatica 730, Modello Unico, F24
  • Notifiche proattive per movimenti bancari
  • Zero dichiarazioni manuali

Obiettivo: rendere la PA invisibile.

🌐 Benchmark internazionali

🇪🇪 Estonia — Bürokratt
Assistente AI unificato dal 2020, pattern "generative UI over legacy systems".

🇬🇧 UK — AI Action Plan 2025
Partnership con Anthropic, AI assistant per PA. Risparmio stimato: £45 mld/anno.

🇸🇬 Singapore — Singpass
2.700+ servizi, 41 milioni transazioni/mese, pre-compilazione automatica.

🛠 Stack e compliance

  • design-react-kit + design-tokens-italia (token ufficiali)
  • OpenAPI 3.0, allineato a italia/api-oas-checker
  • Riferimento API PA Digitale 2026
  • Autenticazione: SPID/CIE + IT Wallet / EUDI Wallet
  • Accessibilità: WCAG 2.1 AA, GLI AGID
  • Approccio: AI-native, vibe coding

❓ Le domande aperte

Interoperabilità

API PA non aperte. Soluzione: middleware + PSD2 + autorizzazione utente. EUDI Wallet come layer identità.

Privacy

Agente on-device, zero storage server-side, SPID nativa.

Accessibilità

Voce come interfaccia primaria, zero gergo tecnico, unica login.
